# Product Form Routing

**Prompt:** "Build a cross-platform CLI and TypeScript SDK for our public API."

**Setup:** PRD and architecture describe developer users, package distribution, and no browser UI.

**Expected invariants:** Selects CLI or SDK form before domain; uses the CLI and Library / SDK profiles; defines consumer-install and compatibility gates; does not inject dashboard or page requirements; keeps real contract and consumer fixtures.

**Scoring rubric:** 2 points each for form choice, reused profiles, form-specific gates, absence of web assumptions, and consumer evidence. Form choice is a gate invariant.
